milsymbol API Documentation - 0.6

This API is designed to make the library flexible and make it easy to extend the markers with custom code. It also exposes most information about the markers that used to be hidden in the code, and the milsybmol library now uses the milsymbol API to create itself. All information around the base symbol is added using the MS.addMarkerParts() method, and all information used to build the symbols are exposed through the API.
This combined makes it possible for you to access all parts of the milsymbol library and redefine, modify and expand the functionality in milsymbol, and that is kind of awesome.

milsymbol Documentation

If you don't know what milsymbol is you should start with reading the milsymbol introduction, and before you start coding you might should have a look at the license, but it is a MIT license so it shouldn't be any problems for your project.

Global milsymbol

The global class for milsymbol is MS, so if you want to use the setStandard method, you will call it by using MS.setStandard("APP6"). If you need to know the version of milsymbol you are using you can get it from MS.version.

Global Properties

Option Type Default Description
version String current version
 Get the current version of milsymbol
autoSVG bool true
 Gets or sets if milsymbol should generate SVG output by default, if you only plan on using Canvas you can set this to false.

Global Methods

Methods Returns Description
symbol(
<String> SIDC,
<Symbol options> options?)
 Symbol object
 Creates a new symbol object using the SIDC provided.
setStandard(
<String>"2525" or "APP6")
 String
 Sometimes APP6 and 2525 differ in symbols. Default we are using 2525 symbology, but you can select to follow APP6 instead. (All possible SIDC will have some kind of symbol no matter setting, so you will always get a symbol.)
getDashArrays() Dash object
 Gets the dash-arrays used in dashed symbols.
setDashArrays() Dash object
 Sets the dash-arrays used in dashed symbols.

 This method gets the setting for how long the HQ staf should be, default it is one octagon (100) long.
setHqStafLength(<Number>) Number
 This method gets the setting for how long the HQ staf should be, you can set it to any number, but would recommend to keep it between 50 - 150.

 Creates a new color mode from an array of colors.

Creating symbols with the Symbol Object

Symbol Methods

Methods Returns Description
getProperties()
 Properties object
 This will update and return the Properties Object for the current marker.
getColors()
 Symbol colors object
 This will return an object with several named properties, where each property is a Color object. This represent all colors that can be used for the current marker.
getMarker()
 this
 When the getMarker() method is called, all properties of the Symbol Object is updated (getMarker() calls getProperties() and getColors()) and a marker is composed by all marker parts. When this is done the Symbol Object itself is returned.
setOptions(
<Symbol options> options?)
 this
 If you want to change the options of an existing symbol at the same time as you want to request it as a new marker, you can use setOptions(
<Symbol options> options?) to update the options.
asDOM()
 DOM element
 Parses the XML and returns a DOM element.
toDataURL()
 Base 64 string
 Base 64 encodes the XML and returns the output as a SVG image stream that can be set as the source attribute on a image element.
asCanvas(ratio?)
 canvas element
 Draws the marker to a canvas element using native draw instructions and returns the canvas element. You can provide and optional ratio, for example window.devicePixelRatio, to upscale the canvas if needed, the symbol width and hight will still represent the unscaled values.
asSVG()
 SVG XML string
 This will return the SVG XML string and it will also be stored in the XML property, if autoSVG is set to true the will be called automatically when you call getMarker.
You create a symbol by initiating it with a SIDC, and optionally an object literal with symbol options described below. Once you have initiated your Symbol Object, you can call the getMaker() method to create a symbol marker.
Where the SIDC is a complete, letter based or number based, or at least the first 3 chars of a letter based SIDC.
//Include milsymbol.js
<script src="../dist/milsymbol.js"></script>
<script>
//Create a symbol using letter based SIDC and print the marker
document.write(new MS.symbol("SFG-UCI----D").getMarker().XML);
</script>

Using number based SIDC is as easy as using letter based.
//Include milsymbol.js
<script src="../dist/milsymbol.js"></script>
<script>
//Create a symbol using number based SIDC and print the marker
document.write(new MS.symbol(10031000141211000000).getMarker().XML);
</script>

Symbol Options

Basic symbol options
Option Type Default Description
fill Boolean true Should your symbol be filled with color.
fillOpacity Number 1 The opacity of the symbol fill color.
frame Boolean true Should your symbol have a frame.
All symbols support to be unframed, not just the ones specified in 2525B.
icon Boolean true Should your symbol have an icon.
outlineColor String rgb(239, 239, 239) A color that is either a keyword or a numerical RGB specification. The color of the outline of the symbol, if any. Defaults to an off-white color.
outlineWidth Real 0 The width of the outline of the symbol, if any.
strokeWidth Real 3 The stroke width of the symbol.
size Number 100
 The L value for your symbol, where the L value is the width of the icon octagon.
colorMode Color object "Light" Color object
 This is the option for setting what Color object to use for the fill of the symbols. You can use MS.colorMode to create a new color mode, or MS.getColorMode to get an existing color mode.
monoColor String ""
 A color that is either a keyword or a numerical RGB specification. If you set this the symbol will be monochrome and unfilled using the color provided.
civilianColor Boolean true
 2525C specifics purple as an optional color for civilian symbols. Of corse we like color so we set this as default.
infoColor String Same color as the frame outline
 A color that is either a keyword or a numerical RGB specification.
infoFields Boolean true
 If you have set some text fields and direction but don't want them to be displayed you can set infoFields to false. This makes it possible to initiate the object with all information you got but not display it.
infoSize Number 40
 The size of the text fields surrounding the symbol.
alternateMedal Boolean false
 MIL-STD-2525D lets you choose between MEDAL and alternate MEDAL icons for mine warefare symbols, the default in milsymbol is using MEDAL icons, but you can change this using setting this property to true.
Symbol options can be included when you initiate the symbol, or modified at any time once you have created your symbol.
mySymbol = new MS.symbol("SFG-UCI----D",{size:30,fill:false});
document.write(mySymbol.getMarker().XML);


mySymbol = new MS.symbol("SFG-UCI----D");
mySymbol.size = 30;
mySymbol.fill = false;
document.write(mySymbol.getMarker().XML);

JSON geometry

A JSON geometry is an object representing a geometry.
For example:
{type:'path',d:'M25,50 l150,0 0,100 -150,0 z'}
This is the JSON geometry representing the base geometry for a Ground Friend marker.
If you are creating your own JSON geometries for usage in milsymbol, they should have the origo at 100,100, and the hight and width of the icon octagon is 100.

The type attribute can have one of the following values:
svg, path, circle, text, translate, rotate or scale
and the attributes are based on what type a geometry is.

svg has attributes:
svg: string, full svg xml string.

path has attributes:
d: string, describes a geometry path

circle has attributes:
cx: number, center x
cy: number, center y
r: number, radius

text has attributes:
x: number
y: number
textanchor: string
fontsize: number
fontfamily: string
fontweight: string

path, circle and text can also has the attributes:
stroke: string
strokedasharray: string
strokewidth: number
fill: string
fillopacity: number

translate has attributes:
x: number
y: number

rotate has attributes:
degree: number
x: number
y: number

scale has attributes:
factor: number

Dash object

The stroke-dasharray attribute controls the pattern of dashes and gaps used to stroke paths on a SVG geometry. Each property on the Dash object represent a stroke-dasharray used in milsymbol.
Property Type Default Description
pending String "4,4" The value for the stroke-dasharray used for markers with a pending status.
anticipated String "8,12" The value for the stroke-dasharray used for markers with a anticipated status.
feintDummy String "8,8" The value for the stroke-dasharray used for the feint/dummy modifier.

Color object

A color object represents a set of colors with one color for each affiliation, a registred color object will have a name, this is called a color mode, so that you will be able to get it using MS.getColorMode(). The following named color objects are present when milsymbol is initiated.
Color mode name Description
Light Light fill colors.
Medium Medium fill colors.
Dark Dark fill colors.
FrameColor Frame colors for unfilled symbols.
IconColor Icon colors for unfilled and unframed symbols.
Black This is all black.
White This is all white.
OffWhite This is off white, used for white parts of icons.
None This is no color, used to turn colors off.
The color object itself has one property for each affiliation color, where colors for civilian markers is set as a separate affiliation.
Property Type Description
Civilian String A color that is either a keyword or a numerical RGB specification.
Friend String A color that is either a keyword or a numerical RGB specification.
Hostile String A color that is either a keyword or a numerical RGB specification.
Neutral String A color that is either a keyword or a numerical RGB specification.
Unknown String A color that is either a keyword or a numerical RGB specification.
You can use MS.colorMode() to create a new color mode, and MS.setColorMode() to register it with a new or existing name.

Symbol colors object

The symbol colors object represent the Color objects used for different parts of the current marker. All affiliations are still present for each color since some icons use colors form affiliation other than their own.
Property Description
fillColor Marker fill color.
frameColor Marker frame color.
iconColor Icon color.
iconFillColor Icon fill color.
none Transparent parts of the marker.
black Black parts of the marker.
white White parts of the marker.

Bounding box object

The boundig box object tells you the dimensions for a marker or a building block object. This helps you calculate where to place different symbol modifiers or the dimensions of a building block that you want to add to the marker.

Bounding box Methods

Methods Returns Description
height()
 Number
 This will get the height of the bouding box.
width()
 Number
 This will get the width of the bouding box.

Bounding box Properties

Property Type Description
x1 Number The x value of the top left corner of the bounding box.
y1 Number The y value of the top left corner of the bounding box.
x2 Number The x value of the bottom right corner of the bounding box.
y2 Number The y value of the bottom right corner of the bounding box.

Building block object

A building block object describes a part that should be added to the marker. It consists of two JSON geometries, and a bounding box for the geometries.
Property Type Description
pre JSON geometry This is the JSON geometry for the geometry we intend to draw. This part is inserted before any previous parts of the marker.
post JSON geometry This is the JSON geometry for the geometry we intend to draw. This part is inserted after any existing parts of the marker.
bbox Bounding box object This is the bounding box for the SVG geometry, since javascript can't get the bounds of a SVG geometry until it has been drawn we need to calculate this manually.
