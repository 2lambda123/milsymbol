Creating a symbol
You create a symbol by initiating it with a SIDC, and optionally an object literal with symbol options described below.
new MilSymbol(<SIDC|String>, <Symbol options> options? )
//Include MilSymbol.js
<script src="../bin/MilSymbol.js"></script>
<script>
//Create a symbol
mySymbol = new MilSymbol("SFG-UCI----D");
</script> Where the SIDC is a complete, letter based or number based, or at least the first 3 chars of a letter based SIDC.

If you want to include options the symbol should be initiated like this, see below for symbol options.
mySymbol = new MilSymbol("SFG-UCI----D",{size:30,fill:false});
Basic Symbol Options
Option Type Default Description
fill Boolean true Should your symbol be filled with color.
frame Boolean true Should your symbol have a frame.
All symbols support to be unframed, not just the ones specified in 2525B.
icon Boolean true Should your symbol have an icon.
strokeWidth Real 3 The stroke width of the symbol.
size Integer 100
 The L value for you symbol. (If L < 10 the symbol will be a filled circle with the diameter of your L, at these sizes symbols won't be readable otherwise.)
colorMode String "Light"
 "Light", "Medium" or "Dark".
monoColor String ""
 Set to any color name "Black"/"Green"/e.t.c. or color value "rgb(255,0,0)".
civilianColors Boolean true
 2525C specifics purple as an optional color for civilian symbols. Of corse we like color so we set this as default.
force2525 Boolean true
 Sometimes APP6 and 2525 differ in symbols, for example 2525 wants VSTOL to be symbolized with the letter L but APP6 want to use the letter V, there are also some differences in fill colors and other. Default we are using APP6 symbology, but if you want to force 2525 symbology, just set this to true. (All possible SIDC will have some kind of symbol no matter setting, so you will always get a symbol.)
infoFields Boolean true
 If you have set some text fields and direction but don't want the to be displayed you can set infoFields to false. This makes it possible to initiate the object with all information you got but not display it.

Getting a symbol
You can get your symbol in two ways. Either use symbolSvg() or symbolMarker().

symbolSvg()
This will return just you symbol without any text feilds. It will return an object with two properies object and textXML. Symbols generated directly from symbolSvg() will have quite large white space around them.

Property Description
object A javascript object representing the SVG XML, and can be used if you want to process your symbol further in your own javascript code.
textXML Is a text representation of the SVG XML, and can be used if you want to write you symbol directly to your document.

Example: mySymbol = new MilSymbol("SFG-UCI----D",{size:30})
document.write(mySymbol.symbolSvg().textXML);

symbolMarker({symbolObject}?)
The symbol marker returned from symbolMarker() is a more complex object that is suitable for when you want to use the symbol as a map marker, the white space around the symbol will also be minimized. The object returned will have six properties: object, textXML, width, height, x and y.

Property Description
object A javascript object representing the SVG XML, and can be used if you want to process your symbol further in your own javascript code.
textXML Is a text representation of the SVG XML, and can be used if you want to write you symbol directly to your document.
width The width of the symbol.
height The height of the symbol.
x The top offset of the insertion point, this will help you position the symbol correctly if you are using it as a map marker.
y The left offset of the insertion point, this will help you position the symbol correctly if you are using it as a map marker.

It's possible to pass an symbolObject from symbolSvg() to this function. If you know that you will update the information around the icon a lot but the SIDC will be the same, the recommendation is to cash the symbolObject and pass it to the function because till will gain you some speed. If you don't pass any object the function will create a new icon from the symbols SIDC.

Example:
//Without cashed symbol
mySymbol = new MilSymbol("SFG-UCI----D",{size:30})
document.write(mySymbol.symbolMarker().textXML);

//With cashed symbol, this is a tiny bit faster if you are going to update your symbol often
mySymbol2 = new MilSymbol("SFG-UCI----D",{size:30})
var cashedSymbol = mySymbol2.symbolSvg();
document.write(mySymbol.symbolMarker(cashedSymbol).textXML);

Updating a symbol
It is possible at any time to update properties for your symbol and request a new rendition.
Example:
var sym = new MilSymbol("SFG-UCA----G",{size:50});
document.write(sym.symbolMarker().textXML);
document.write("<br>");
sym.size = 29;
sym.uniqueDesignation = "NAME";
document.write(sym.symbolMarker().textXML);
